# 电影评价预测模型

基于豆瓣电影数据的传统机器学习预测模型项目（计算思维期末大作业 - 要求2）

---

## 📖 项目简介

### 背景与目标

本项目是 **计算思维期末大作业的第二部分**，旨在利用传统机器学习方法，基于电影的上映前特征（档期、百度搜索指数、片长、细分类型等），预测电影的最终表现：

**预测任务**：
1. **评价人数预测**：预测电影会获得多少评价（热度/票房代理指标）
2. **豆瓣评分预测**：预测电影的质量评分（0-10分）

### 最新模型性能（v2.0 - 40 特征）

| 任务 | 模型 | CV R² | 测试 R² | MAE | MAPE |
|------|------|:---:|:---:|:---:|:---:|
| 评价人数 | 随机森林 | **0.75** | **0.51** | 48,554 | 116% |
| 评分 | 随机森林 | **0.53** | **0.55** | 0.80 | 16.7% |

> v1.0（7 特征）测试 R² 分别为 0.37 和 0.34。通过增加百度指数衍生特征和细分类型标签，性能提升约 50-60%。

### 技术特点

- ✅ 使用 **Scikit-learn** 实现传统机器学习算法
- ✅ 对比 **线性回归** 和 **随机森林** 两种算法
- ✅ 完整的数据预处理流程（缺失值处理、特征缩放、对数变换、one-hot 编码）
- ✅ 严格的模型评估（交叉验证、多指标评估、可视化分析）
- ✅ 输出 RMSE、MAE、R²、MAPE 等评估指标
- ✅ 生成预测vs实际散点图、特征重要性图表

## 🚀 快速开始

### 前置条件

1. **已完成数据采集**：确保 `../doban-data/data/final/china_all.xlsx` 文件存在（含百度指数 5 项特征 + 细分类型标签）
2. **Python 环境**：Python 3.8 或更高版本
3. **虚拟环境**（推荐）：建议使用虚拟环境隔离依赖

### 安装步骤

#### 1. 进入项目目录

```powershell
cd "E:\计算思维期末大作业\movie-prediction"
```

#### 2. 安装依赖

```powershell
pip install -r requirements.txt
```

### 运行流程

**完整执行顺序**（必须按顺序运行）：

```powershell
python src/1_data_preparation.py
python src/2_feature_engineering.py
python src/3_train_models.py
python src/4_evaluate_models.py
```

#### 步骤1: 数据准备
- 从 `../doban-data/data/final/china_all.xlsx` 加载原始数据（16750 条，29 列）
- 筛选有有效百度指数的记录（6019条）
- 处理片长字段（解析文本，转换为分钟数，中位数填充缺失值）
- **对豆瓣细分类型做 one-hot 编码**（29 个类型标签如 类型_剧情、类型_动作…）
- 分割训练集和测试集（80%/20%）
- 输出：40 列特征（5 档期 + 5 百度指数 + 1 片长 + 29 类型），2 列目标

#### 步骤2: 特征工程
- 对数值特征进行标准化（StandardScaler）：5 项百度指数 + 片长
- 档期和类型特征保持 0/1 不变
- 对评价人数进行对数变换：`log(评价人数 + 1)`

#### 步骤3: 训练模型
训练 4 个模型：线性回归 / 随机森林 ×（评价人数 / 评分），各做 5 折交叉验证

#### 步骤4: 评估模型
在测试集上评估，计算 RMSE、MAE、R²、MAPE，生成 6 张图表

## 📊 数据说明

### 数据来源

- **原始数据源**：`../doban-data/data/final/china_all.xlsx`
- **总样本数**：16750条中国地区电影记录
- **有效样本数**：6019条（有有效百度指数的记录，占比35.9%）

### 特征详解

#### 输入特征 (X) - 共 40 个特征

**1. 档期特征（5 个二元特征）**

| 特征名 | 含义 | 取值 |
|--------|------|------|
| `是否春节档` | 春节档期标识 | 0或1 |
| `是否国庆档` | 国庆档期标识 | 0或1 |
| `是否五一档` | 五一档期标识 | 0或1 |
| `是否暑期档` | 暑期档期标识（7-8月） | 0或1 |
| `是否普通周末` | 非节假日的周五/六/日 | 0或1 |

**2. 百度指数（5 个连续特征）**

| 特征名 | 含义 | 说明 |
|--------|------|------|
| `上映前7天百度搜索指数均值` | 映前热度平均值 | 反映基础关注度 |
| `百度指数标准差` | 热度波动程度 | 值越大热度越不稳定 |
| `百度指数趋势斜率` | 热度趋势方向 | 正值=热度上升，负值=下降 |
| `百度指数最大值` | 映前峰值热度 | 反映话题引爆程度 |
| `百度指数最小值` | 映前底线热度 | 反映最低关注度 |

**3. 片长（1 个连续特征）**

| 特征名 | 含义 | 数据范围 |
|--------|------|----------|
| `片长` | 电影时长 | 30-300分钟 |

**4. 细分类型（29 个二元特征）**

从豆瓣原始 `类型` 字段（如"悬疑 / 惊悚 / 犯罪"）做 one-hot 编码生成：
动作、喜剧、剧情、爱情、科幻、悬疑、犯罪、动画、奇幻、冒险、惊悚、恐怖、战争、武侠、古装、纪录片、短片、历史、家庭、音乐、传记、儿童、歌舞、运动、情色、同性、灾难、西部、黑色电影

#### 预测目标 (Y) - 2 个回归任务

| 目标变量 | 含义 | 处理方式 |
|----------|------|----------|
| `评价人数` | 豆瓣评分用户数 | `log(x+1)` 对数变换后训练，预测时反变换 |
| `评分` | 电影平均评分（0-10） | 不转换，直接使用原始值 |

## 🤖 模型说明

### 算法选择

#### 线性回归 (Linear Regression)
基线模型，可解释性强，每个特征系数直接反映影响方向。

#### 随机森林回归 (Random Forest Regressor)
集成学习，100棵树，max_depth=10。能捕捉非线性关系，输出特征重要性。

---

## 📈 评估指标说明

| 指标 | 含义 | 解读 |
|------|------|------|
| **RMSE** | 均方根误差（原单位） | 越小越好 |
| **MAE** | 平均绝对误差（原单位） | 越小越好 |
| **R²** | 决定系数（-∞, 1] | 越接近1越好，0=等同于猜均值 |
| **MAPE** | 平均绝对百分比误差 | 如 20% = 平均偏差 20% |

## ⚠️ 注意事项

1. **数据依赖**：确保先运行 `doban-data` 项目的完整流程（步骤1-6），且最终数据包含细分类型列和 5 项百度指数特征
2. **值域警告**：`上映前7天百度搜索指数均值` 为 `pd.NA` 的记录会被自动过滤；片长文本"未提取到片长"会转为 NaN 后用中位数填充
3. **图表中文**：已自动配置微软雅黑字体
4. **动态特征列**：模型会根据实际 CSV 列名自适应，不硬编码特征列表
