问题定义

阐述选题背景与分析目标

选题背景

电影产业是文化消费的重要组成部分，一部电影的市场表现（票房、口碑）受到多重因素影响： 上映档期、前期宣传热度、影片内容质量与时长等。对片方和发行方而言，如果能在上映前就对一部电影的 市场反响做出预判，便能更科学地安排排片计划、宣发预算和档期选择。

五个核心研究问题

  • 评分与受关注度（评价人数）存在怎样的关系？
  • 不同档期的电影表现是否有显著差异？
  • 上映前百度搜索指数能否预测最终受关注度？
  • 能否用客观特征预测评价人数与评分？
  • 大模型能挖掘哪些超越类型标签的内容特征？

三个层面的分析目标

  • 描述层面：刻画市场整体分布特征
  • 预测层面：构建回归模型预测热度与口碑
  • 智能层面：大模型语义打标 + 档期题材分析

数据通过自建爬虫从豆瓣采集，覆盖 27 个类型， 经清洗、去重、关联百度指数后形成最终数据集 china_all.xlsx。

传统机器学习模型构建与评估

Scikit-learn · 线性回归 vs 随机森林（对应要求 2）

两个回归任务

  • 任务 A：预测评价人数（市场热度代理，反映票房潜力）
  • 任务 B：预测豆瓣评分（质量代理，反映口碑）

约 40 个输入特征

  • 档期特征（5个）：春节/国庆/五一/暑期/普通周末
  • 数值特征（6个）：百度指数均值/标准差/斜率/最大/最小 + 片长
  • 类型特征（约28个）：豆瓣细分类型 one-hot 编码

两个对比模型

线性回归（基线）：假设线性关系，可解释性强， 用于验证问题是否存在简单线性结构。

随机森林（主力）：100 棵决策树集成， 能捕捉非线性关系与特征交互，对异常值更鲁棒。

n_estimators=100 max_depth=10 5折交叉验证 8:2 划分 log1p 变换

大模型 API 调用与智能分析

DeepSeek API · 剧情简介语义打标（对应要求 3）

设计思路

传统类型标签（动作、爱情）较粗糙，无法刻画"催泪""合家欢""沉重压抑"这类贴近观众情感体验的特征。 调用 DeepSeek（deepseek-v4-flash） 对剧情简介做语义分析， 自动打上细粒度标签，填补传统特征工程的空白。

工程实现要点

  • 批处理：每批 10 部合并为一次请求
  • 强制 JSON：response_format 保证可解析
  • 指数退避重试：tenacity，最多 3 次
  • 断点续传：每 100 部存检查点
  • 8 线程并发：耗时 60 分钟 → 9 分钟

39 个预定义标签体系（6 大类）

情感类

催泪温馨感动虐心治愈

风格类

喜剧黑色幽默荒诞文艺商业

主题类

主旋律青春励志家庭历史战争都市

节奏类

烧脑紧张刺激轻松搞笑沉重压抑

提示词设计（System + User 双角色）

System Prompt

  • 角色定位：专业电影内容分析助手
  • 严格约束：只能从 39 个标签中选，不创造新标签
  • 限定输出：只返回 JSON

User Prompt

  • 传入 10 部电影标题+简介（截断 500 字）
  • 每部选 1-3 个最符合的标签
  • 温度 0.3，输出更稳定

实验结论

六项核心发现
1 · 市场关注度高度集中
评价人数呈长尾分布，少数爆款占据绝大多数关注，必须配合对数变换与非线性模型才能有效建模。
2 · 前期热度是最强预测信号
上映前 7 天百度搜索指数均值是预测评价人数最重要的特征，前期热度能有效转化为实际关注。
3 · 档期效应真实存在
热门档期关注度显著更高，但各档期评分差异小——档期带来的是流量红利，而非质量背书。
4 · 特征工程是决定性因素
特征 7→40 个后 CV R² 从 0.37 升至 0.75，提升超 100%。特征质量比算法调参更重要。
5 · 集成模型显著优于线性模型
随机森林全面超越线性回归，说明数据中存在大量非线性关系与特征交互。
6 · 大模型弥补结构化特征盲区
传统模型无法"读懂"剧本，大模型能从简介提取情感基调，二者互补构成完整决策支持。

心得体会

困难解决 · 传统模型 vs 大模型对比

过程中遇到的四个困难

  • 中文字体乱码：matplotlib 默认不支持中文，用 FontProperties 显式指定微软雅黑路径解决。
  • pandas 填充失效（最隐蔽）：CoW 机制下 df[col].fillna(inplace=True) 作用在临时副本被丢弃，改赋值写法解决。
  • 线性回归 R² 负数：目标变量极度右偏，爆款主导损失函数，用 log1p + 随机森林解决。
  • API 调用慢：串行 60 分钟 → 8 线程并发 9 分钟（独立客户端 + 互斥锁）。

传统模型 vs 大模型

维度传统机器学习（随机森林）大模型（DeepSeek）
输入数据结构化数值/类别特征非结构化自然语言文本
核心优势可量化评估、特征可解释理解语义、处理语言
主要局限无法理解内容含义成本高、不可量化评估
可解释性强（特征重要性）弱（黑盒）
边际成本训练后接近零按调用量持续付费
本项目作用预测热度/评分挖掘情感标签

两者不是替代而是互补关系：传统模型回答"这部电影会有多火"， 大模型回答"这部电影是什么情绪基调"，二者结合才构成完整的决策支持系统。 选择正确的工具，比调参更重要。

总结

通过这次大作业，我完整走了一遍"数据采集 → 清洗 → 探索性分析 → 特征工程 → 建模预测 → 大模型智能分析"的 数据科学全流程。最大的收获是：遇到报错不能慌，要静下心定位根因——很多看似玄学的 bug， 背后都有清晰的技术原因，搞明白一个真正的根因，比盲目试错十次都有价值。
